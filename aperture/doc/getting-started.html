Getting Started

To obtain Aperture, you can download the latest release at SourceForge.net or checkout and compile the latest version directly from CVS. While the project is still in alpha stage, the latter will most likely give you the best code to work with.

Download the latest release

The latest Aperture release can be downloaded at http://sourceforge.net/project/showfiles.php?group_id=150969.

Unzip the zip or tar.gz file to a location of your choice and open the created folder.

The lib directory contains the Aperture jar files as well as third-party jar files and their licenses.

For running Aperture you will at least need aperture-<version>.jar. The aperture- examples-<version>.jar file is only needed when using the Aperture example classes. The aperture-test-<version>.jar file is normally not needed. You will also need the openrdf-model and openrdf-util jar files, as they are used by the Aperture APIs.

Most other jar files are used by Extractor and Crawler implementations. If you don't use them, you will probably not need them.

The src.zip file contains all Aperture source code, including examples and JUnit tests.

The bin directory contains batch files and shell scripts for starting the GUI versions of the examples. These include a file system crawler, an IMAP mail folder crawler and a file inspector, showing you the output of Aperture's MIME type identification and full-text and metadata extraction.

Note that there are also various commandline examples whose code is a lot simpler to understand.

The minimum Java version to run Aperture is Java 1.4.2, but it runs fine with Java 5 and 6 as well.

Checkout from CVS

First, you should read http://sourceforge.net/cvs/?group_id=150969 about getting the latest Aperture sources from CVS.

Once checked out, you can use the Ant build script in Aperture's root directory to build the project. Notable targets are:

  • compile: compiles all classes, excluding unit tests and example classes (use compile.examples and compile.test for that).
  • jar: creates the core Aperture jar file (likewise, use jar.examples and jar.test for the other two).
  • dist: creates and gathers all jar files and javadocs in build/dist. All unit tests are also applied.
  • release: combines the dist results with all other required files (scripts, licenses, ...) and puts them in build/release.
  • package: creates .zip and .tar.gz files of the release results (used to created the dowloadable releases).
  • test: run all JUnit tests.
  • clean: removes all generated files.

The minimum Java version to run Aperture is Java 1.4.2, but it runs fine with Java 5 and 6 as well.

The precise minimum Ant requirements are currently unknown; the build files have been tested with Ant 1.6.5, older versions may or may not work.
